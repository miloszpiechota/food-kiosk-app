# Personas

## Persona 1: Kiosk Customer

### Description
A customer using a self-service kiosk terminal in a restaurant to browse the menu, choose items, review the basket, and place an order.

### Goals
- Find products quickly
- Browse products by category
- Search for specific products
- Configure meals without confusion
- Review selected products before checkout
- Complete the order with minimal friction

### Needs
- A clear and easy-to-use kiosk interface
- Large, readable, touch-friendly controls
- Fast navigation between categories and products
- Guided meal selection and product personalization when required
- A visible basket and order summary
- Accessible interaction patterns, including strong contrast and clear focus states

### Pain Points
- Confusing navigation
- Slow or cluttered menu browsing
- Unclear order summary
- Difficulty using the interface with accessibility needs

## Persona 2: Restaurant Admin

### Description
A restaurant administrator or staff member responsible for monitoring and managing customer orders coming from the kiosk system.

### Goals
- View incoming orders clearly
- Track order status
- Manage the order workflow efficiently
- Reduce mistakes in order handling

### Needs
- A simple order management view
- Clear order details
- Reliable order status handling
- Confidence that payment-related order states are trustworthy

### Pain Points
- Missing or unclear order information
- Inefficient order handling flow
- Difficulty tracking current order state
- Risk of relying on incomplete payment confirmation logic

## Notes
These personas describe the intended users of the system during the discovery phase. They represent product direction and not implemented functionality.
