# Glossary

## Admin Panel
The protected area of the application used by restaurant administrators to view and manage orders.

## Administrator
A user with authorized access to the admin panel for managing restaurant operations in the system.

## Basket
The temporary collection of menu items selected by a customer before checkout. Also commonly called a cart.

## Category
A grouping of menu items that helps customers browse the menu more easily.

## Checkout
The step in the ordering flow where the customer reviews the order and proceeds to payment.

## Customer
A restaurant guest using the kiosk interface to browse the menu and place an order.

## Kiosk
A self-service terminal-style interface used by customers to order food without staff assistance.

## Menu
The list of available food or drink items offered by the restaurant through the kiosk application.

## MVP
Minimum Viable Product. The smallest useful version of the application that delivers the core ordering and admin management flow.

## Order
A customer purchase request created through the kiosk system. An order may exist before payment is fully confirmed.

## Order Status
The operational state of an order from the restaurant perspective, such as new, in progress, ready, or completed.

## Payment Status
The state of payment verification, such as pending, paid, failed, cancelled, or awaiting confirmation.

## Stripe Checkout
Stripe's hosted payment flow used to handle customer payment in test mode for this project.

## Test Mode
A non-production payment environment used for development and testing without real financial transactions.

## Touch-Friendly UI
An interface designed so users can interact easily through tapping, including clear controls and sufficiently large interactive targets.

## Webhook
A backend endpoint that receives event notifications from another service, such as Stripe payment updates, for verification and processing.
